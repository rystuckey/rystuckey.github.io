## I. Introduction

AI has become a major part of how students learn and how developers write software.  In a software engineering course like ICS 314, AI can be especially useful because the work often involves learning new tools, debugging unfamiliar errors, writing code quickly, understanding frameworks, and explaining technical concepts.  However, AI can be detrimental to learning if it is used as a replacement for understanding instead of as a tool to support learning.

During ICS 314, I used AI throughout the semester in different ways.  In the first half of the semester, I mainly used ChatGPT, especially for WODs and timed assignments where the code had to be close to perfect in a short amount of time.  Later in the semester, I started using GitHub Copilot inside VS Code more often because it was directly integrated into my coding environment.  Copilot helped with writing code, completing repetitive patterns, debugging, and working through environment issues involving tools like Next.js, Prisma, Vercel, TypeScript, and ESLint.

Overall, AI was helpful as a learning tool and improve my skills through its guidance. It could give me a starting point, explain an error, or help me move faster, but I still had to check the output, understand whether it fit the assignment, and fix anything that did not work.  My experience in ICS 314 showed me that AI is best used when it acts like a coding assistant to help with completing tasks under my own guidance or a tutor to help me learn the skills I need.

## II. Personal Experience with AI

### 1. Experience WODs

I used ChatGPT during the Experience WODs, especially earlier in the semester when I was still getting used to the course workflow and the speed expected for WODs.  Because in-class WODs are timed and require a perfect solution, I used AI to help me understand the structure of what I needed to build before starting and had it help me practice during these experience WODs.

During these WODs I used prompts along the lines of "Here are the instructions, explain in steps what I need to create, and what tools/methods should I use here?"

### 2. In-class Practice WODs

For in-class Practice WODs, I used AI less as a direct code generator and more as a way to understand the problem. Since these were practice assignments, the main value was learning how to approach the task. I would ask ChatGPT to explain the concepts behind the WOD, such as React component structure, props, Bootstrap layout, or JavaScript/TypeScript syntax.  I would use prompts similar to the experience WODs but more asking for assistance rather than code generation.

### 3. In-class WODs

For graded in-class WODs, I used AI more for code generation under my instruction because these assignments were timed and performance-based.  Earlier in the semester, I relied on ChatGPT more heavily because I felt pressure to produce working code quickly. I would use it to create boilerplate templates, autofill content based on WOD instructions, and debug code.

I would paste instructions into AI to give it an understanding of the project, and used it as a pair programming tool to get a working solution within the time limit.

AI was useful in this situation because it could often identify simple mistakes faster than I could, especially syntax errors, missing imports, incorrect component names, or problems with state/props. However, the limitation was that the answer was only useful if I already understood enough to judge it. In a timed WOD, blindly copying AI output could waste time if the output did not match the assignment or introduced a new problem.


### 4. Essays

I used ChatGPT for several ICS 314 essays, especially for planning, outlining, and creating first drafts that I could edit into my own style.  At first I would write essays fully on my own, but I moved to using ChatGPT to automate some parts of writing essays.  In this case, AI helped me organize my thoughts and connect course concepts to my actual project experience.

I would prompt it with the essay instructions, my thoughts and ideas, and ask it for structure and ideas on how to set up the essay and sample text that I can build on myself.

This was useful because writing technical essays can be difficult when I know the general idea but do not know how to structure it. C hatGPT helped me turn scattered ideas into paragraphs with a clear flow.  However, I still needed to edit the writing so it sounded like me and matched my real experience.  If I used the AI output directly without editing, the essay would sound too generic and would not fully reflect what I personally learned.

My final project involved tools like Next.js, TypeScript, React, Prisma, PostgreSQL, Vercel, and GitHub.  I used AI when working through tasks related to deployment, database setup, course data, page configuration, authentication, and review functionality. Later in the semester, Copilot helped write code patterns, suggest fixes, and speed up repetitive implementation work.

A representative use of Copilot was letting it autocomplete code, database query patterns, or page logic while I worked in VS Code. ### 6. Learning a Concept / Tutorial

I used AI as a tutor when I needed a concept explained in a simpler way. This was especially useful for topics like TypeScript, design patterns, Bootstrap, ESLint, Prisma, and Next.js routing. Sometimes documentation is technically complete but hard to understand quickly. ChatGPT helped translate documentation-style explanations into step-by-step explanations.

This was useful because I could ask follow-up questions immediately. For example, if I did not understand a TypeScript error, I could ask ChatGPT to explain why the type mismatch was happening and how to think about it. The limitation was that AI sometimes oversimplified concepts or gave examples that did not match the exact structure of our course projects.

### 7/8. Answering a Question in Class, in Discord, or asking Smart Questions

I did not ask any questions in the discord over the semester, and questions in class were asked on my own using my own ideas to better my own understanding.  I didn't see a necessary use case for AI in this sense, but AI could be used as a replacement to ask questions when not in class, but that is likely not the scope of this question.

### 9. Coding Example

I used AI to generate small coding examples when I needed to understand a syntax pattern or API. This was helpful for learning because it gave me a smaller version of the problem before applying it to the full project.

This could be applied to Bootstrap components, TypeScript props, mapping over an array, Prisma queries, or a form submission pattern.  These examples were helpful because they were smaller and easier to understand than the full project code. However, I had to be careful because examples often leave out details like validation, error handling, authentication, or project-specific imports.

### 10. Explaining Code

I used AI to explain code when I was reading unfamiliar code or trying to understand how a piece of the project worked. This was especially helpful in a team project because not all code was written by me. If I needed to understand a file before editing it, AI could summarize what the code was doing.  I could ask the AI what the code was doing and explain what is happening step by step and it would give me a good explanation.

This was useful because it helped me understand the intent of the code faster. It was especially helpful for larger files or files involving database access, authentication, or forms. The downside was that AI could miss project context, so I could not rely only on the explanation. I still had to read the actual code and test changes myself.

### 11. Writing Code

I used AI for writing code throughout the semester, but how I used it changed over time. Early in the semester, I used ChatGPT more often by copying code into the chat and asking for a solution or fix. Later in the semester, I used GitHub Copilot more because it worked directly inside VS Code and could suggest code while I was typing.

Copilot was especially useful for repetitive code patterns, such as creating components, filling in form fields, writing TypeScript structures, or following patterns that already existed in the project. A representative Copilot use was writing the beginning of a component or function and letting Copilot suggest the next few lines.

The benefit was that I could work faster and avoid getting stuck on small syntax details. The cost was that Copilot sometimes generated code that looked good but did not fit our exact schema, imports, or authentication/session setup. I had to review and edit the output instead of accepting it automatically.

### 12. Documenting Code

I used AI less for documenting code than for writing or debugging code. However, I did use AI for writing explanations around the project, including essays, portfolio entries, and summaries of what the project did. This is related to documentation because it helped me explain technical work in a way that someone else could understand.

### 13. Quality Assurance

AI was very useful for quality assurance, especially for debugging and fixing errors. I used ChatGPT and Copilot to help identify what was wrong with code, explain error messages, and suggest fixes. This was helpful for ESLint, TypeScript, Next.js, Prisma, and Vercel issues.

I could give AI the code and the error I was facing and ask "Why does this error happen and can you troubleshoot the issue?"

This was one of the best uses of AI because debugging often requires noticing small details. AI could point out missing imports, wrong types, incorrect function signatures, or formatting issues. However, AI was not always correct. Sometimes it fixed one error but introduced another. Because of that, I still had to run the code, check the build, and make sure the final result actually worked.

### 14. Other Uses in ICS 314 Not Listed

One additional way I used AI was for planning and reflection. Near the end of the semester, I used ChatGPT to help organize my final portfolio updates, project entry, effort estimation essay, software engineering essay, and this AI reflection essay. This was not the same as using AI to complete a coding task. It was more about organizing my thoughts and making sure I addressed the assignment requirements.

## III. Impact on Learning and Understanding

AI had a major impact on how I learned in ICS 314. It helped me get unstuck faster, especially when I was dealing with timed WODs, unfamiliar syntax, or confusing errors. Instead of spending a long time searching through documentation or Stack Overflow, I could ask ChatGPT to explain the issue directly. This made it easier to keep moving when I felt blocked.

At the same time, AI also challenged my learning because it was tempting to rely on it too much. If AI gave me a working answer, I still had to ask myself whether I understood why it worked. This mattered especially during WODs because the point was not just to submit working code, but to build the ability to solve similar problems independently.

I think AI improved my learning when I used it to explain concepts, debug errors, or give me a starting point. It was less helpful when I asked for a complete solution without first understanding the problem. The most valuable use of AI was when I treated it like a tutor: asking follow-up questions, comparing its answer to the course material, and testing the code myself.

## IV. Practical Applications

Outside of individual ICS 314 assignments, AI has practical value in real software engineering work. My final project showed this clearly. Manoa CourseWise had multiple tools and systems working together, including Next.js, TypeScript, Prisma, PostgreSQL, GitHub, and Vercel. AI helped with the kind of problems that happen in real projects: debugging environment issues, understanding framework errors, generating code patterns, and explaining unfamiliar parts of the stack.

I have also used AI outside this class for technical and work-related tasks, including planning, writing, debugging, and organizing complex information. In a real software engineering environment, AI can help developers move faster, especially when working with unfamiliar tools or repetitive code. However, the developer still has to be responsible for the final result. AI does not replace testing, code review, security awareness, or understanding the system.

In practical use, I think AI is strongest as an assistant for narrowing down possibilities. For example, if a Vercel deployment fails, AI can suggest common causes to check. If a TypeScript error appears, AI can explain what the type system is complaining about. If a developer needs to write a repeated code pattern, Copilot can speed up the process. These are useful applications, but they still require human judgment.

## V. Challenges and Opportunities

The biggest challenge I encountered with AI was that it could be confidently wrong. It often gave answers that sounded reasonable, but they did not always match the exact assignment, course style, or project setup. This was especially true for newer tools or specific environments like Next.js, Prisma, and Vercel. Sometimes the issue depended on version-specific behavior, and AI did not always know the exact version or configuration.

Another challenge was deciding how much help was too much. For timed WODs, AI could help me move faster, but if I relied on it too heavily, I might not fully develop the skill myself. This made me realize that using AI well requires discipline. I needed to use it to support learning rather than replace learning.

The opportunity is that AI can make software engineering education more interactive. Instead of waiting for help, students can ask questions, get explanations, and debug problems in real time. In the future, courses could teach students not just whether they can use AI, but how to use it responsibly: how to write good prompts, how to verify output, how to cite or disclose AI use, and how to avoid copying code without understanding it.

## VI. Comparative Analysis

Traditional learning methods, such as lectures, readings, documentation, and working through assignments manually, are still important. They provide structure and force students to build their own understanding. Without that foundation, AI output is harder to evaluate because the student may not know whether the answer is correct.

AI-enhanced learning is different because it is interactive and immediate. If I do not understand a TypeScript error, I can ask for an explanation. If I need a small coding example, I can generate one. If I am stuck on a project bug, I can ask for possible causes. This makes learning feel more responsive than only reading documentation or searching online.

However, AI-enhanced learning can also be less reliable. Documentation and course materials are usually designed around the exact course expectations, while AI gives general answers. Because of that, I think the best approach is a combination of both. Traditional methods provide the foundation, and AI helps with practice, explanation, debugging, and iteration.

## VII. Future Considerations

In the future, I think AI will become a normal part of software engineering education and professional development. Tools like ChatGPT and GitHub Copilot are already useful for explaining code, generating examples, writing boilerplate, and debugging. As these tools improve, students and developers will probably use them more often.

However, I do not think AI removes the need to learn software engineering fundamentals. If anything, it makes fundamentals more important. A developer needs to understand the problem well enough to know whether AI output is correct. They need to test the code, check security, understand the architecture, and maintain the project over time.

For future classes, I think it would be useful to include more explicit instruction on how to use AI well. This could include examples of good prompts, bad prompts, how to verify AI-generated code, and how to document AI use. AI is not going away, so learning how to use it responsibly should be part of modern software engineering education.

## VIII. Conclusion

My use of AI in ICS 314 changed throughout the semester. At first, I mainly used ChatGPT to help with WODs, especially because they were timed and required working code quickly. Later, I started using GitHub Copilot more inside VS Code for writing code, debugging, and working through project issues. By the final project, AI had become a regular part of my workflow, especially for planning, debugging, code generation, and understanding tools like Next.js, Prisma, Vercel, and TypeScript.

The biggest lesson I learned is that AI is powerful, but it is not a replacement for understanding. It can help me move faster, explain confusing concepts, and suggest solutions, but I still need to verify the output and understand how it fits the project. In ICS 314, AI was most useful when I used it as a tutor, assistant, and debugging partner. It was least useful when I treated it like something that could solve the whole problem for me.

Overall, I think AI should be integrated into software engineering education, but in a thoughtful way. Students should be encouraged to use AI to learn, experiment, and debug, while also being responsible for understanding and verifying their work. My experience in ICS 314 showed me that AI can make me a more efficient developer, but only if I stay actively involved in the process.
